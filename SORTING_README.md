# Email Sorting Tool

This tool analyzes email markdown files generated by the email-to-markdown exporter and categorizes them into three groups:

- **🗑️ To Delete** - Low-value emails that can be safely deleted
- **📝 To Summarize** - Emails that can be condensed to save space
- **💾 To Keep** - Important emails that should be preserved in full

## Features

### Intelligent Categorization
- **Content Analysis**: Examines email subject, body, and metadata
- **Sender Analysis**: Identifies important vs. promotional senders
- **Attachment Detection**: Preserves emails with important attachments
- **Age-Based Scoring**: Recent emails are prioritized over old ones
- **Type-Based Scoring**: Direct emails are valued more than newsletters

### Comprehensive Reporting
- Detailed statistics by category
- Sender and date analysis
- JSON report for further processing
- Console summary for quick review

### Customizable Rules
- Configurable keywords for each category
- Adjustable thresholds for email age and size
- Custom sender lists (keep/delete)
- Email type weighting system

## Installation

No additional dependencies needed beyond what's required for the main email-to-markdown project.

## Usage

### Basic Usage

```bash
# Sort emails in a directory
python sort-emails.py /path/to/email/export

# With custom configuration
python sort-emails.py /path/to/email/export --config sort_config.json

# Verbose output
python sort-emails.py /path/to/email/export --verbose

# Custom report name
python sort-emails.py /path/to/email/export --report my_report.json
```

### Example with Email Export Directory

```bash
# Assuming you have exported emails to ~/emails/export
python sort-emails.py ~/emails/export

# This will create a sort_report.json file in the same directory
```

## Configuration

The tool uses a JSON configuration file to define sorting rules. A sample configuration is provided in `sort_config.json`.

### Configuration Options

```json
{
    "delete_keywords": ["newsletter", "promotion", "marketing"],
    "delete_senders": ["no-reply@", "marketing@"],
    "delete_subjects": ["Your weekly digest", "Special offer"],
    
    "summarize_keywords": ["meeting", "update", "status"],
    "summarize_max_length": 5000,
    
    "keep_keywords": ["contract", "invoice", "urgent"],
    "keep_senders": ["contracts@", "billing@"],
    "keep_subjects": ["Contract", "Invoice"],
    
    "recent_threshold_days": 30,
    "old_threshold_days": 365,
    
    "small_email_threshold": 500,
    "large_email_threshold": 10000,
    
    "keep_with_attachments": true,
    
    "type_weights": {
        "newsletter": -2,
        "mailing_list": -1,
        "group": 0,
        "direct": 1,
        "unknown": 0
    }
}
```

### Customizing for Your Needs

1. **Add your own keywords**: Extend the keyword lists with terms specific to your email patterns
2. **Adjust thresholds**: Change the age and size thresholds based on your email volume
3. **Sender management**: Add domains or email patterns to keep/delete lists
4. **Type weighting**: Modify the scoring system for different email types

## How It Works

### Scoring System

Each email is assigned a score based on multiple factors:

- **Email Type**: Newsletters (-2), Direct emails (+1)
- **Age**: Recent emails (+2), Old emails (-1)
- **Size**: Very short (-1), Very long (+1)
- **Attachments**: With attachments (+2)
- **Keywords**: Delete keywords (-1 each), Keep keywords (+2 each)
- **Senders**: Delete senders (-3), Keep senders (+3)
- **Content**: Important content in body (+2)

### Category Determination

Based on the score and specific indicators:

- **Keep**: Score ≥ 2, has important keywords, has attachments, or is very long
- **Delete**: Score ≤ -2, is newsletter, has delete keywords, or is from delete senders
- **Summarize**: Everything else (moderate score, average importance)

### Analysis Process

1. **File Discovery**: Recursively scans the directory for `.md` files
2. **Frontmatter Extraction**: Parses YAML metadata from each email
3. **Content Analysis**: Examines subject, body, sender, date, and attachments
4. **Scoring**: Calculates a comprehensive score for each email
5. **Categorization**: Assigns each email to the appropriate category
6. **Reporting**: Generates statistics and detailed reports

## Best Practices

### Regular Maintenance

Run the sorting tool periodically to keep your email archive clean:

```bash
# Monthly maintenance
python sort-emails.py ~/emails/export --report monthly_report_$(date +%Y-%m).json
```

### Integration with Email Export

Combine with the main email export tool:

```bash
# Export and sort in one workflow
python export_emails.py
python sort-emails.py ~/emails/export
```

### Automated Cleanup

Use the JSON report to automate cleanup processes:

```python
import json

# Load the report
with open('sort_report.json', 'r') as f:
    report = json.load(f)

# Process delete recommendations
for email in report['categories']['delete']:
    # Your cleanup logic here
    print(f"Would delete: {email['file']}")
```

## Advanced Usage

### Custom Configuration

Create multiple configuration files for different sorting strategies:

```bash
# Aggressive cleanup
python sort-emails.py ~/emails/export --config aggressive_config.json

# Conservative approach
python sort-emails.py ~/emails/export --config conservative_config.json
```

### Batch Processing

Process multiple email directories:

```bash
for dir in ~/emails/*; do
    echo "Processing $dir"
    python sort-emails.py "$dir" --report "${dir}/sort_report.json"
done
```

### Integration with Other Tools

Use the JSON output with other tools:

```bash
# Convert to CSV for spreadsheet analysis
jq -r '.categories.keep[] | [.file, .subject, .sender, .score] | @csv' sort_report.json > keep_emails.csv

# Get statistics
jq '.summary' sort_report.json
```

## Troubleshooting

### Common Issues

**No emails found**: Ensure the directory contains `.md` files with proper YAML frontmatter

**Parsing errors**: Check that your email files have valid YAML frontmatter format

**Configuration errors**: Verify your JSON config file is valid

### Debugging

Use verbose mode to see detailed processing:

```bash
python sort-emails.py ~/emails/export --verbose
```

## Technical Details

### Algorithm

1. **File Analysis**: Each email file is parsed to extract metadata and content
2. **Scoring**: Multiple factors contribute to a comprehensive score
3. **Categorization**: Rules-based system with score thresholds
4. **Statistics**: Comprehensive metrics collection
5. **Reporting**: Structured output for human and machine consumption

### Performance

- Processes ~100 emails per second (depending on file size)
- Memory efficient (processes one file at a time)
- Scales well with large email archives

### Dependencies

- Python 3.7+
- PyYAML (already required by main project)
- Standard library modules only
